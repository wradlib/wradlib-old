wradlib.adjust.AdjustAdd¶

class wradlib.adjust.AdjustAdd(obs_coords, raw_coords, nnear_raws=9, stat='median', mingages=5, minval=0.0, mfb_args=None, Ipclass=<class wradlib.ipol.Idw>, **ipargs)¶

Gage adjustment using an additive error model.

First, an instance of AdjustAdd has to be created. Calling this instance then does the actual adjustment. The motivation behind this performance. In case the observation points are always the same for different time steps, the computation of neighbours and inverse distance weights only needs to be performed once.

AdjustAdd automatically takes care of invalid gage or radar observations (e.g. NaN, Inf or other typical missing data flags such as -9999). However, in case e.g. the observation data contain missing values, the computation of the inverse distance weights needs to be repeated in __call__ which is at the expense of performance.

Note

Inherits from AdjustBase

For a complete overview of parameters for the initialisation of adjustment objects, as well as an extensive example, please see wradlib.adjust.AdjustBase.
